# 뱀의 지혜

## Ouroboros에서 배우는 Claude Code 기반 Agentic Coding

> 자기 꼬리를 삼키는 뱀은 반복하지 않는다 — 진화한다.

---

### 이 책에 대하여

이 책은 오픈소스 프로젝트 [Ouroboros](https://github.com/Q00/ouroboros)(약 22만 줄의 Python, 1,100여 개 파일)의 **문서와 소스 코드를 해부하여** 추출한 Agentic Coding 원칙을 담고 있습니다. Ouroboros는 "명세 우선 AI 워크플로 엔진"이지만, 이 책의 관심사는 도구 자체가 아닙니다. 이 도구가 **비결정적인 AI 에이전트를 신뢰 가능한 소프트웨어 생산 수단으로 길들이기 위해 코드로 구현해 놓은 설계 결정들** — 그것이 우리가 채굴할 광맥입니다.

Ouroboros의 모든 원칙은 관념이 아니라 **작동하는 코드**로 존재합니다. 모호도 게이트는 `AMBIGUITY_THRESHOLD = 0.2`라는 상수이고, 리워드 해킹 방어는 `REWARD_HACKING_VETO_THRESHOLD = 0.7`이라는 거부권이며, "막히면 다르게 생각하라"는 4가지 정체 감지 알고리즘과 5개 페르소나 프롬프트입니다. 숫자로 구현된 철학은 검증 가능하고, 검증 가능한 철학은 배울 수 있습니다.

각 장은 같은 구조를 따릅니다:

1. **원리** — Ouroboros가 어떤 문제를 어떻게 풀었는가 (실제 코드·수치·인용 포함)
2. **일반화** — 도구에 독립적인 Agentic Coding 원칙으로 추상화
3. **Claude Code 적용** — CLAUDE.md, 계획 모드, 서브에이전트, hooks, skills로 오늘 바로 실천하는 법

### 다섯 원칙 요약 (책 전체의 지도)

```
             ┌─────────────────────────────────────────────┐
             │  I. 명세    "명확해지기 전에는 만들지 마라"     │
             │             (입구 게이트: Ambiguity ≤ 0.2)    │
             └──────────────────┬──────────────────────────┘
                                ▼
             ┌─────────────────────────────────────────────┐
             │  II. 검증   "완료는 승인이 아니다"             │
             │             (비용 사다리: $0 → $$ → $$$)      │
             └──────────────────┬──────────────────────────┘
                                ▼
             ┌─────────────────────────────────────────────┐
             │  III. 기록  "기록 없는 것은 일어나지 않았다"    │
             │             (이벤트 소싱, 재현, 드리프트 측정)  │
             └──────────────────┬──────────────────────────┘
                                ▼
             ┌─────────────────────────────────────────────┐
             │  IV. 진화   "반복하지 말고 진화하라"           │
             │             (출구 게이트: Similarity ≥ 0.95)  │
             └──────────────────┬──────────────────────────┘
                                ▼
             ┌─────────────────────────────────────────────┐
             │  V. 경계    "자율성에는 경계를 세워라"          │
             │             (예산·워치독·fail-closed)         │
             └─────────────────────────────────────────────┘
```

두 개의 수학적 관문이 전체를 감쌉니다: **명확해지기 전에는 만들지 말고(Ambiguity ≤ 0.2), 안정되기 전에는 진화를 멈추지 말라(Similarity ≥ 0.95).** 나머지 모든 기계 장치는 이 두 관문 주변의 방어 시설입니다.

### 이 책을 읽는 법

- **바쁘다면**: [8장 패턴 카탈로그](./08-catalog.md)부터. 체크리스트만으로도 실천을 시작할 수 있습니다.
- **원리가 궁금하다면**: 1장부터 순서대로. 각 원칙은 앞 원칙 위에 쌓입니다.
- **Claude Code 사용자라면**: 각 장 말미의 "Claude Code 적용" 절과 [7장](./07-claude-code.md)이 실전 요리법입니다.

### 출처 표기

본문의 인용과 수치는 모두 Ouroboros 저장소의 실제 파일에서 가져왔으며 `src/ouroboros/...` 형태로 경로를 표기합니다. 원문이 영어인 인용은 필요 시 한국어로 옮기고 원문을 병기했습니다.
